## 待办任务（三期：用户体验与内容分发）

### TASK-201: 头像系统（Avatar）
- **优先级**: 高
- **状态**: ✅ 已完成
- **描述**:
  - 在 `User` 模型中复用/增强 `image` 字段，支持管理员在后台上传自定义头像。
  - 若未上传，使用 Gravatar 作为 fallback（基于 email 的 MD5 哈希）。
  - 头像展示位置：Header 用户菜单、文章页作者署名区、关于页、OpenGraph 动态图（后续 TASK-206）。
- **验收标准**:
  - [ ] 后台 `/admin/profile` 或设置页可上传头像，支持裁剪预览。
  - [ ] 头像文件保存到 `public/uploads/avatars/`（或 OSS，本期优先本地）。
  - [ ] 文章页底部增加「作者简介卡片」，展示头像、昵称、一句话简介。
  - [ ] 未上传头像时，自动请求 Gravatar；Gravatar 不存在时回退到默认占位图。

### TASK-202: 站点通知系统（Site Notifications）
- **优先级**: 高
- **状态**: ✅ 已完成
- **描述**:
  - 提供两种通知形态：**全局顶部横幅（Banner）** 和 **文章内嵌提示块（Callout）**。
  - Banner 用于站点级公告（如「本站即将维护」），支持关闭（`localStorage` 记录已关闭的 ID，24h 内不再显示）。
  - 文章内嵌 Callout 用于文章级更新说明（如「2026-04-20 更新：修正了某处错误」），在 Markdown 中通过 `:::note` / `:::warning` 语法渲染（使用 remark 插件或自定义解析）。
- **验收标准**:
  - [x] 新增 `Announcement` 模型（`id`, `content`, `type: banner|toast`, `link?`, `isActive`, `startAt`, `endAt`, `createdAt`）。
  - [x] 管理后台新增「公告管理」页面，可增删改查、设置有效期。
  - [x] 前端 Layout 自动拉取活跃 Banner，渲染在 Header 下方；用户点击关闭后写入 `localStorage`。
  - [x] `MarkdownRenderer` 支持 `:::note` / `:::warning` / `:::tip` 语法，渲染为带图标的提示块。

### TASK-203: RSS 订阅增强

- **状态**: ✅ 已完成

- **验收标准**:

  - [x] 文章详情页和 Footer 增加 RSS 图标链接。
  - [x] 新增 `/feed.json` 路由，输出 JSON Feed v1 格式。

### TASK-204: 文章搜索（全文 + 快捷键）
- **优先级**: 高
- **状态**: ✅ 已完成
- **描述**:
