# 首页改版 AI 提示词模板

## 用途

用于约束 AI 生成首页布局与样式，避免回到低效的“大卡片 + 大 Hero”方案。

## 可直接复用的提示词

你是资深前端设计工程师。请改造我的博客首页，目标是“阮一峰式高实用性 + 现代克制审美”。

硬性要求：
1. 桌面端使用双栏布局：主列表 8/12，右侧效率栏 4/12；移动端自动单列。
2. 顶部仅保留极简标题行：页面标题 + 文章总数 + 归档入口。
3. 主列表必须是高密度单行条目（日期 + 标题 + 轻量元信息），禁止摘要卡片。
4. 右侧效率栏固定顺序：
   - 站内搜索
   - 分类/标签快速入口（按数量排序，最多 8 个）
   - 年份归档（近年汇总）
   - 最近更新（最近 5 篇标题）
5. 右栏桌面端 sticky，移动端下沉到底部简化展示。
6. 样式遵循现有设计系统：克制、轻阴影、低装饰密度、左对齐、可扫描性优先。

明确禁止：
- 不要 Hero 大区块
- 不要全宽大卡片列表
- 不要标题 + 摘要 + 标签的重复信息堆叠
- 不要华而不实动效

交付格式：
1. 先给简短结构说明（不超过 8 行）
2. 再给可运行代码（优先复用现有组件）
3. 最后给验收清单（响应式、可访问性、交互、回归影响）

验收标准：
- 桌面首屏可见更多文章条目，右侧不空白
- 文章列表信息密度明显高于旧版
- 分类/标签/归档/最近更新入口都可点击
- 移动端不拥挤，阅读路径清晰
